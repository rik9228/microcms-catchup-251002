# CMSとは（10 min）

## 1. CMSの基本

### 🔸 CMSってなに？
**CMS（Content Management System）** は、  
専門的な知識がなくてもサイトの内容を更新・管理できる仕組みのことです。

たとえば、会社のニュースやお知らせ、ブログ記事などを  
毎回HTMLで更新するのは大変ですよね。  
そこで、CMSを使うと**ブラウザ上の管理画面から簡単に投稿や編集ができる**ようになります。

---

### 🔸 どんな仕組み？

1. 管理画面で記事を登録・編集  
2. 記事の内容はデータベースに保存  
3. Webサイトはデータベースから記事を読み込み、ページとして表示  

この流れで、**技術者でなくても更新できる**仕組みを実現しています。

---

### 🔸 CMSを使うメリット

- 🧑‍💻 **コーディング不要で更新できる**  
  HTMLを直接編集しなくてもOK。  
- ⚡ **更新作業がスムーズ**  
  ブラウザ上で文章や画像を差し替えるだけ。  
- 🧩 **デザインと内容を分けて管理できる**  
  見た目と中身を分離しているため、保守や修正がしやすい。  

---

## 2. WordPressのような「従来型CMS」

CMSの中でも有名なのが **WordPress** です。  
WordPressは「記事の管理」から「ページの表示」までを**ひとつの仕組みで完結**させています。

### 📊 WordPressの流れ
1. 管理画面で記事を投稿  
2. 投稿内容がデータベースに保存  
3. テーマファイル（テンプレート）を使って記事を表示  

すべてをWordPressの中で処理するため、  
**導入が簡単で、初心者にも扱いやすい**のが特徴です。

---

## 3. ヘッドレスCMSとは？

近年は、もう少し柔軟にサイトを作りたいというニーズが増えています。  
そこで登場したのが **ヘッドレスCMS** です。

---

### 🔸 「ヘッドレス」の意味
「ヘッド（見た目）」＝ 表示部分  
「レス」＝ 取り外す  

つまり、  
> **“見た目の部分をCMSから切り離した仕組み”**  
がヘッドレスCMSです。

---

### 🔸 どう違うの？

| 項目 | 従来型CMS（例：WordPress） | ヘッドレスCMS |
|------|-----------------------------|----------------|
| 管理 | 管理画面で記事を登録 | 同じく管理画面で記事を登録 |
| 表示 | CMS内のテーマで表示 | API経由でデータを取得し、別の仕組みで表示 |
| 特徴 | 管理と表示が一体化 | 管理と表示が完全に分離 |

### 🔸 イメージ図

【WordPressの場合】
```mermaid

┌────────────┐
│  管理画面   │  ←（記事を投稿・編集）
└────┬───────┘
     │
     ▼
┌────────────┐
│ データベース │  ←（記事データを保存）
└────┬───────┘
     │
     ▼
┌──────────────────┐
│ テーマ/テンプレート │  ←（デザイン・表示処理）
└────┬─────────────┘
     │
     ▼ 
┌────────────┐
│  Webサイト  │  ←（ユーザーが閲覧）
└────────────┘
```

【ヘッドレスCMSの場合】
```mermaid

┌────────────┐
│  管理画面   │  ←（記事を投稿・編集）
└────┬───────┘
     │
     ▼
┌────────────┐
│ データベース │  ←（記事データを保存）
└────┬───────┘
     │
     ▼
┌────────────┐
│    API     │  ←（記事データを外部に提供）
└────┬───────┘
     │
     ▼
┌──────────────┐
│ フロントエンド │  ←（Webサイトやアプリなど、自由な技術で表示）
└──────────────┘
```

### 🔸 メリット・デメリット

|  | ヘッドレスCMS | WordPress |
|---|----------------|------------|
| メリット | ・表示が高速<br>・自由なデザインや構成が可能<br>・セキュリティが高い | ・すぐ使える<br>・プラグインで機能を拡張できる |
| デメリット | ・フロントエンド開発が必要<br>・導入コストがやや高い | ・サイトが重くなりやすい<br>・攻撃対象になりやすい |

---

### 🔸 どんなときに使う？

- Webサイトやアプリを**複数媒体で共通管理**したい  
- **表示デザインを自由に作りたい**（React / Vue / Astroなど）  
- WordPressほど大規模でなくてもいいけど、**管理画面は欲しい**  

---

## 5. まとめ

| ポイント | 内容 |
|-----------|------|
| CMSとは | サイトの内容を管理できる仕組み |
| WordPress | 管理と表示がセットになった従来型 |
| ヘッドレスCMS | 管理と表示を分離し、自由に開発できる |
| microCMS | 日本語対応のヘッドレスCMS。小規模〜中規模に最適 |

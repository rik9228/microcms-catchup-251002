# CMSとは（10 min）

## 1. CMSの基本

### 🔸 CMSってなに？
**CMS（Content Management System）** は、  
専門的な知識がなくてもサイトの内容を更新・管理できる仕組みのことです。

たとえば、会社のニュースやお知らせ、ブログ記事などを  
毎回HTMLで更新するのは大変ですよね。  
そこで、CMSを使うと**ブラウザ上の管理画面から簡単に投稿や編集ができる**ようになります。

---

### 🔸 どんな仕組み？

1. 管理画面で記事を登録・編集  
2. 記事の内容はデータベースに保存  
3. Webサイトはデータベースから記事を読み込み、ページとして表示  

この流れで、**技術者でなくても更新できる**仕組みを実現しています。

---

### 🔸 CMSを使うメリット

- 🧑‍💻 **コーディング不要で更新できる**  
  HTMLを直接編集しなくてもOK。  
- ⚡ **更新作業がスムーズ**  
  ブラウザ上で文章や画像を差し替えるだけ。  
- 🧩 **デザインと内容を分けて管理できる**  
  見た目と中身を分離しているため、保守や修正がしやすい。  

---

## 2. WordPressのような「従来型CMS」

CMSの中でも有名なのが **WordPress** です。  
WordPressは「記事の管理」から「ページの表示」までを**ひとつの仕組みで完結**させています。

### 📊 WordPressの流れ
1. 管理画面で記事を投稿  
2. 投稿内容がデータベースに保存  
3. テーマファイル（テンプレート）を使って記事を表示  

すべてをWordPressの中で処理するため、  
**導入が簡単で、初心者にも扱いやすい**のが特徴です。

---

## 3. ヘッドレスCMSとは？

近年は、もう少し柔軟にサイトを作りたいというニーズが増えています。  
そこで登場したのが **ヘッドレスCMS** です。

### サービス一覧（日本・国内）

- **microCMS**（日本発、手軽に始められる）
- **Kuroko**（日本発、開発者向け）
- **Newt**（日本発、エンタープライズ向け）※2024年6月サービス終了予定
- **Contentful**（グローバルに展開、豊富な機能）
- **Strapi**（オープンソース、自分でホスティング可能）

etc

---

### 🔸 「ヘッドレス」の意味
「ヘッド（見た目）」＝ 表示部分  
「レス」＝ 取り外す  

つまり、  
> **“見た目の部分をCMSから切り離した仕組み”**  
がヘッドレスCMSです。

---

### 🔸 どう違うの？

| 項目 | 従来型CMS（例：WordPress） | ヘッドレスCMS |
|------|-----------------------------|----------------|
| 管理 | 管理画面で記事を登録 | 同じく管理画面で記事を登録 |
| 表示 | CMS内のテーマで表示 | API経由でデータを取得し、別の仕組みで表示 |
| 特徴 | 管理と表示が一体化 | 管理と表示が完全に分離 |

```
     │
     ▼ 
┌────────────┐

```

【ヘッドレスCMSの場合】
```
┌────────────┐
│  管理画面   │  ←（記事を投稿・編集）
└────┬───────┘
     │
     ▼

└────┬───────┘
     │
     ▼

└────┬───────┘
     │
     ▼
┌──────────────┐
│ フロントエンド │  ←（Webサイトやアプリなど、自由な技術で表示）
└──────────────┘
```

### 🔸 メリット

- **サイトの一部だけでも利用できる**
  - 従来のCMSの場合、お知らせや製品ページなど、頻繁に更新したいページだけにCMSを導入することは難しいとされていた。ヘッドレスCMSは、必要な箇所にAPIを通じて該当する箇所を呼び出し、既存ページに反映させることが可能。
- **フロントエンドの自由度が高い**
  - ヘッドレスCMSではフロントエンド部分が独立しているという特徴がある。フロントエンドとバックエンドが完全に独立しているので、それぞれが言語などの配慮する必要がなく、開発や改修を行うことができる。
- **セキュリティ面が頑強**
  - 従来のCMSは、管理画面と表示部分が一体化しているため、攻撃を受けるリスクが高い。ヘッドレスCMSは、管理画面と表示部分が分離されているため、攻撃を受けにくい。

など

### 🔸 デメリット

- **導入・運用コストが高い**
  - 従来のCMSは、管理画面と表示部分が一体化しているため、導入や運用が比較的容易である。一方、ヘッドレスCMSは、管理画面と表示部分が分離されているため、導入や運用にコストがかかる場合がある。（テンプレートやスターターキットの充実で近年は低減傾向）
- **技術的な知識が必要**
  - ヘッドレスCMSは、APIを通じてデータを取得し、フロントエンドで表示する必要があるため、技術的な知識が必要となる場合がある。

## 5. まとめ

| ポイント | 内容 |
|-----------|------|
| CMSとは | サイトの内容を管理できる仕組み |

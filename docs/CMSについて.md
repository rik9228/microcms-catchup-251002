## ヘッドレスCMについて

**→ ヘッド（＝ビュー／フロントエンド）を持たないバックエンドの機能だけを持つCMS。**

### 特徴

- 表示速度が速い
- サイトの一部だけをCMS化できる
- セキュリティが高い　。。。etc

| 項目 | ヘッドレスCMS | WordPress |
| --- | --- | --- |
| **メリット** | ・表示速度が速い<br>・マルチデバイスに対応できる<br>・サイトの一部だけをCMS化できる<br>・フロントエンドの改修を行いやすい<br>・セキュリティが高い | ・コンテンツ管理と表示部分の両方をカバーできる<br>・コストを抑えられる<br>・テーマやプラグインで簡単にカスタマイズ可能 |
| **デメリット** | ・フロントエンドの開発者が必要<br>・WordPressで簡易にできていたことを自前で実装する必要がある（プレビュー機能など）<br>・動的な機能には外部ツール導入が必要 | ・フロントエンドはバックエンドに合わせて作成・改修する必要がある<br>・セキュリティに弱く、運用に手間がかかる |
| **向いている企業** | ・Webサイトやアプリの共通コンテンツを一元管理したい企業<br>・企業独自のデザインや機能開発を行いたい企業 | ・コンテンツ管理から表示までを一括で管理したい企業<br>・WordPressのテーマやプラグインを活用し、独自開発をあまり必要としない企業 |

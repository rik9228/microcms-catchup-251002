# CMSとは - 10 min

## 1. CMSの基本

### 🔸 基本概念

**CMS（Content Management System）** は、  
専門的な知識がなくてもサイトの内容を更新・管理できる仕組みのことです。

たとえば、会社のニュースやお知らせ、ブログ記事などを  毎回HTMLで更新するのは大変ですよね。  
そこで、CMSを使うと**ブラウザ上の管理画面から簡単に投稿や編集ができる**ようになります。

[参考](https://www.shift-jp.net/blog/cms/)

---

### 🔸 どんな仕組み？

1. 管理画面で記事を登録・編集  
2. 記事の内容はデータベースに保存  
3. Webサイトはデータベースから記事を読み込み、ページとして表示  

この流れで、**技術者でなくてもコンテンツの更新ができる**仕組みを実現しています。

---

### 🔸 CMSを使うメリット

- 🧑‍💻 **コーディング不要で更新できる**  
  HTMLを直接編集しなくてもOK。  
- ⚡ **更新作業がスムーズ**  
  ブラウザ上で文章や画像を差し替えるだけ。
- 📅 **複数人で管理できる**  
  権限設定を行い、担当者ごとに編集範囲を分けられる。など

---

## 2. WordPressの立ち位置

CMSの中でも有名なのが **WordPress** です。  
WordPressは「記事の管理」から「ページの表示」までを**ひとつの仕組みで完結**させています。

### 📊 WordPressの流れ

1. 管理画面で記事を投稿  
2. 投稿内容がデータベースに保存  
3. テーマファイル（テンプレート）を使って記事を表示  

すべてをWordPressの中で処理するため、  
**導入が簡単で、初心者にも扱いやすい**のが特徴です。

## 3. ヘッドレスCMSとは？

近年は、もう少し柔軟にサイトを作りたいというニーズが増えています。  
そこで登場したのが **ヘッドレスCMS** です。

### サービス一覧（国内・海外）

- **microCMS**
- Kuroko
- Newt ※2024年6月サービス終了予定
- Contentful
- Strapi

参考サイト：
[https://scr.marketing-wizard.biz/utilities/headlesscms-major4-comparison]

etc

---

### 🔸 「ヘッドレス」の意味
「ヘッド（見た目）」＝ 表示部分  
「レス」＝ 取り外す  

つまり、  
> **“見た目の部分をCMSから切り離した仕組み”**  
がヘッドレスCMSです。

-> 普通のCMS（たとえばWordPress）は、
記事を投稿したら そのままページとして表示 までやってくれます。

でもヘッドレスCMSは、「記事を管理する部分」だけを担当して、
フロントの部分は開発者が自由に設計・実装 します。

---

### 🔸 どう違うの？

| 比較項目         | WordPress               | ヘッドレスCMS（例：microCMS）                     |
| ------------ | ----------------------- | ---------------------------------------- |
| **構造**       | 「記事管理」＋「表示」までを一体で行う     | 「記事管理」と「表示」を分離する                         |
| **サイト表示**    | PHPでサーバー上でページを生成（動的）    | APIでデータを受け取り、JSフレームワークなどで表示              |
| **実装方法や構成の自由度**      | WordPressのテーマ構造に縛られる    | フロントを自由に構築できる（Next.js / Astro / Nuxt など） |
| **更新の仕組み**   | 管理画面 → DB → テンプレート で即反映 | 管理画面 → API経由でデータ取得 → 表示側が反映              |
| **向いてるシーン** | ブログや企業サイトをすぐ作りたい人向け     | 開発者が自由にデザインや構成を決めたい場合に向く                 |


### 💬 もう少し感覚的に言うと…

WordPressは「家（Webサイト）」と「中身（記事）」が一体になった“完成済みの家”
→ テンプレートを選んで、家具（記事）を置くだけで住める状態。

ヘッドレスCMSは「家具管理システム」だけを提供しているイメージ。
→ “どんな家を建てるか（デザイン・構造）”は開発者が自由に決める。

### 🔸 メリット

- **部分的に利用できる**
  - ヘッドレスCMSは、必要な箇所にAPIを通じて該当する箇所を呼び出し、既存ページに反映させることが可能。
- **セキュリティ面が強固**
  - ヘッドレスCMSは、管理画面と表示部分が分離されているため、従来型CMSに比べ攻撃を受けるリスクが低い。
- **サイトパフォーマンスの向上が図れる**
  - ヘッドレスCMSは、APIを通じて必要なデータだけを取得するため、ページの読み込み速度が向上しやすい。※実装の仕方にもよる
- **フロントエンドの自由度が高い**
  - ヘッドレスCMSではフロントエンド部分が独立しているという特徴がある。フロントエンドとバックエンドが完全に独立しているので、それぞれが言語などの配慮する必要がなく、開発や改修を行うことができる。

など

### 🔸 デメリット

- **導入・運用コストが高い**
  - 従来のCMSは、管理画面と表示部分が一体化しているため、導入や運用が比較的容易である。一方、ヘッドレスCMSは、管理画面と表示部分が分離されているため、導入や運用にコストがかかる場合がある。（テンプレートやスターターキットの充実で近年は低減傾向な印象）
- **技術的な知識が必要**
  - ヘッドレスCMSは、APIを通じてデータを取得し、フロントエンドで表示する必要があるため、技術的な知識が必要となる場合がある。
- **動的な機能には別途外部ツールが必要**
  - お問い合わせフォームやコメント機能など、動的な機能を実装する場合、別途外部ツールやサービスを利用する必要がある場合がある。

など

---

### 🧠 まとめ

- CMSの役割は「記事などのコンテンツデータを管理すること」
- WordPressは「管理＋表示」をまとめて行う“オールインワン”型
- ヘッドレスCMSは「データ管理だけ」に特化し、“表示や組み方”は自由設計
